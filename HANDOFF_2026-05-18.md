## ⚠️ Important framing — the benchmark measures Groq+prompt, NOT full MIRA

`tools/answer_quality_benchmark.py` calls Groq directly via httpx with a
custom diagnostic prompt and fixture-side KB chunks. It bypasses:

- The UNS confirmation gate (`mira-bots/shared/uns_resolver.py`)
- The intent classifier (`mira-bots/shared/guardrails.py`)
- The FSM (`mira-bots/shared/fsm.py`)
- The real RAGWorker (Open WebUI retrieval)
- The production prompt template (`prompts/diagnose/active.yaml`)
- `citation_compliance.py` enforcement
- `Supervisor.process()` entirely

This is a **scaffolding baseline**, not the May 21 acceptance gate.
The score below describes what a hypothetical "Groq + good prompt + good
chunks" system produces — useful as a north-star for what the full engine
should achieve, but not what the bot currently does.

The proper benchmark is `python -m tests.conversation_suite.harness
--mode=live --filter=tag:demo_may21`, which uses the real Supervisor.
That requires Open WebUI reachable from the test machine. I verified
that the VPS Open WebUI is not externally exposed (`mira-core:8080`
is docker-internal only, port 8080 not open on Tailscale or public IP),
so I couldn't run it from Charlie. **This is the must-do-before-May-21
follow-up.** See "MUST DO BEFORE MAY 21" section below.

## Risk / what to watch

- **Per-fixture variance is real.** At Groq temperature 0.2, fixture scores swing ±0.5–1.5 points run-over-run. The aggregate 3.86 is the trustworthy signal; don't gate on any single fixture. The pytest test correctly checks aggregate, not per-fixture.
- **Citation rate is 60%, below the 90% spec target.** This is a documented gap, not a regression. The fixes here move it up but the prompt-side `[Source:]` discipline needs to land in the production engine prompt template (`prompts/diagnose/active.yaml`) to be permanent.
- **Q9 ("Is the PLC seeing the sensor?") is hard with single-turn benchmark.** Without session context the bot can't know whether it should clarify or give a generic checklist. The spec acknowledges this tension (`diagnosis.md` §3); operator may want to add a multi-turn version of Q9 in a follow-up.
- **Anthropic in `tests/eval/judge.py`.** The judge calls Claude when the response was generated by Groq, per cross-model objectivity. This is eval-only, not the production cascade — does not contradict the "no Anthropic in production" rule. Worth knowing if you audit the judge.
