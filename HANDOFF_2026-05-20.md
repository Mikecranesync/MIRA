## Decisions made (and why)

- **Sidebar "Feed" → `/feed`, not renamed `event-log`.** Mike's brief said primary should be Feed; `/feed` route already exists. `event-log` route stays reachable but drops out of the sidebar (can be wired under Admin later).
- **"CMMS" relabels existing `workorders` key**, doesn't add a new route.
- **"Scan", "Admin", "Settings"** added as net-new nav entries pointing at existing routes (`/scan`, `/admin`, `/integrations`). "Settings" pointing at `/integrations` is a stopgap — separate settings page can land later.
- **Mock pages: in-page early-return** (vs route-group layout) — minimal-diff approach, preserves the existing client-component structure of each page.
- **`/quickstart` cite-or-refuse system prompt.** The standard UNS gate (`docs/specs/maintenance-namespace-builder-spec.md`) doesn't apply because there's no plant context. The grounding contract becomes: if no KB chunk supports the answer, refuse and offer signup. Documented in ADR-0014.
- **Middleware excludes `/quickstart` and `/api/quickstart/*`** so the auth gate doesn't bounce anonymous visitors.
- **No new manufacturer index.** Re-used the existing `knowledge_entries.manufacturer` column + GIN BM25 index via the existing `retrieveManualChunks` helper.

## Late catches

- **basePath fetch bug.** First snap of `/quickstart` looked fine but the fetches were root-absolute (`/api/quickstart/*`) while routes serve at `/hub/api/quickstart/*` because `next.config.ts` sets `basePath="/hub"`. The page rendered identically because the empty-manufacturers fallback fires on 404 OR DB-unset. Caught by advisor review. Fixed in commit `d27ee0b3` (use `API_BASE` from `@/lib/config`). curl confirms `/hub/api/quickstart/manufacturers` → 200.
- **Initial screenshots were taken before the basePath fix** — the page renders the same either way (empty fallback), so the existing PNGs are still representative. Re-snap from staging once DB is wired.
- **CI on PR #1467** was kicked at push of `d27ee0b3`. Run: https://github.com/Mikecranesync/MIRA/actions/runs/26146399218. Migration Order Check passed; Lint, Static Analysis, DeepEval, etc. still pending at handoff time. If 🔴 IMPORTANT comments land, `bash scripts/pr_self_fix.sh 1467` is the prescribed one-shot autofix.

## Risks / things to watch

1. **`QUICKSTART_TENANT_ID` unset on prod = demo-tenant data leaks to public page.** Set this env in `factorylm/prd` before deploy. There's a comment in both new routes pointing at this.
2. **Cite-or-refuse depends on the model honoring the system prompt.** Standard Groq → Cerebras → Gemini cascade. If the corpus is sparse for a manufacturer, the model may either refuse (good) or hallucinate (bad). Test with `tests/golden_*.csv` adversarial inputs once staging is live.
3. **`docker-compose.pathb.yml` still has `mira-sidecar`.** That's intentional (edge bundle), but worth knowing if you ever sunset Path-B.
4. **Build emits "middleware deprecated, use proxy"** — Next.js 16.x renamed the convention. Not breaking yet; track for a future hub PR.
5. **Sidebar order for "Settings → /integrations" may surprise users** who expect Settings to be a dedicated page. Filed mentally as a P1 follow-up.
