# MiraTagStream — Ignition 8.3 gateway timer resource (GROUND TRUTH)

This is the **gateway-proven** on-disk format for the FactoryLM collector's
timer script, captured from the CV-101 bench gateway (Ignition 8.3.4,
2026-07-03) after it was verified loading and firing. It is the template the
collector resource pack (`docs/plans/2026-07-03-ignition-collector-resource-pack.md`)
must package.

## Why this exists

The June 2026 install staged the timer as `ignition/event-scripts/timer/<Name>/code.py`
— the **8.1-era layout, silently ignored by 8.3** (the timer never fired once,
June → July). The real 8.3 contract, extracted from the gateway's own
`common.jar` (`ScriptConfig$TimerScript`, `TimerKey$Serializer`) and then
proven live:

```
ignition/timer/<TimerName>/            ← resource type: module "ignition", type "timer"
├── handleTimerEvent.py                ← "def handleTimerEvent():" + body indented 4 spaces
└── resource.json                      ← this file; attributes below
```

`resource.json` attributes (see the file):

| key            | meaning                                   | CV-101 value |
|----------------|-------------------------------------------|--------------|
| `delay`        | period in ms (long)                       | 2000         |
| `fixedDelay`   | `false` = Fixed **Rate**, `true` = Fixed Delay | false   |
| `sharedThread` | `false` = Dedicated thread                | false        |
| `enabled`      | timer enabled                             | true         |

Notes proven on the bench:

- `lastModificationSignature` is **not required** — hand-written resources load.
- The gateway does **not** rescan externally-written project files; restart the
  Ignition service to load them.
- `handleTimerEvent.py` is generated by wrapping the repo source
  `ignition/gateway-scripts/tag-stream.py` (see the pack builder plan) —
  do not hand-edit it; the wrapper is byte-derivable from the source.
