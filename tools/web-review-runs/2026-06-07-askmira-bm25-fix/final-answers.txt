--- CE10 answer ---
The fault code CE10 on the AutomationDirect GS10 variable frequency drive indicates a Modbus timeout, which means the host controller (in this case, the Allen-Bradley Micro820 PLC) is not continuously transmitting data to the GS10 during Modbus communication [Source: AutomationDirect — Fault Code Table]. To resolve this issue, the following steps can be taken: 
1. Verify the host controller is transmitting data continuously to the GS10.
2. Check the Modbus communication connection and cable integrity.
3. Review the P09.03 setting for transmission timeout detection time.
4. Send a Reset command to clear the error code by writing 2 to register 0x2002 [Source: AutomationDirect — Fault Code Table].
