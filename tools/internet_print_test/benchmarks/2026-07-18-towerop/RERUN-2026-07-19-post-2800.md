## What genuinely improved (structural, verified)

1. **Evidence flows**: `ocr_items` 0 → 18–75 on all 7 pipeline cases (184/156 even on the R2 table pages); `ocr_source=tesseract` provenance on every turn. The deployment smoking gun from the baseline (spine inert) is **closed**.
2. **Classification strengthened**: ELECTRICAL_PRINT at 0.85 (baseline 0.60) on pipeline cases.
3. **The keep-alive regime works in production conditions**: autoeval fired **3 live P0s** during the run (c02 `invented_tags`+`degenerate_enumeration`, c04 `unsupported_state_claim`, c09 `degenerate_enumeration`) — tonight's #2800/#2802 machinery catching tonight's real garbage.
4. **$0 end-to-end** (ZTA-clean); OpenAI interpreter still 429 quota-dead on every pipeline case (+5–8 s tax), identical to baseline conditions.

## Where the bottleneck moved

From **"no evidence"** (baseline) to **"synthesis can't use the evidence"**: gemma-3n-E4B drops correct upstream detections (c04's S7.1 was IN its own vision_description), denies text whose garbled fragments sit in its own `ocr_items` (c05), and launders OCR noise into prose (c03/c09). Three levers, in leverage order:

1. **Routing first (unchanged blocker, 5/12 cases)**: R1 caption-gate vocabulary + R2 visual-first routing (#2713 — c11/c12 are now premium fixtures with OCR truth attached).
2. **Evidence contract in the theory prompt**: require verbatim quoting of used OCR tokens, mark non-matching/garbled tokens unverified, and forbid "not labeled" claims when `ocr_items` is non-empty without checking — plus an autoeval `false_absence_claim` candidate rule (c05's reply contradicted its own evidence array; today's rules didn't flag c05/c07 — that class has no rule yet).
3. **Model quality**: this run is the empirical case for **PR-F** (paid interpreter re-enable — credits + sign-off) and/or the `OCR_MODEL_LANE` benchmark: the free VL floor reads prints structurally but cannot ground fine text.
