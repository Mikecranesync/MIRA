## Next round — CONFIRMED levers (code-verified mechanisms, not hypotheses)

Each lever carries its mechanism receipt: this run's evidence plus the responsible code path, verified 2026-07-19 against main `2ff4e78e1`. Ordered by effect per unit of work.

**L1 — Narrow the wiring carve-out (restores c04; trivial fix).**
`mira-bots/telegram/bot.py:1177` — `_try_print_translator_reply` declines the turn when `wiring_intake.parse_wiring_intent(caption).kind != "none"`. The parser (`shared/wiring_intake.py:246–258`) returns THREE kinds — `intake`, `question`, `none` — and the carve-out was written for intake ownership (its own comment: "an explicit wiring-intake command") but fires on `question` too. c04's caption ("Which PLC input is the … switch … **wired to** …") parses as a wiring question → the print rung declines pre-vision (capture: `handled=false`, no classification — first run ever; both priors answered c04). **Fix:** decline only on `kind == "intake"`. **Regression test:** c04's caption verbatim + a print photo must classify. **Effect:** wiring-phrased print questions — a large share of real technician phrasings — stop dying silently.

**L2 — Dense-table override must outrank NAMEPLATE unit-vocabulary (restores c10; small, fixture-backed).**
`mira-bots/shared/workers/vision_worker.py` — `NAMEPLATE_OCR_FIELDS` (≥3 unit fields like hp/kW/Hz/volts in OCR ⇒ NAMEPLATE "regardless of how the vision model describes it", ~line 103) is checked BEFORE `DENSE_TABLE_OCR_THRESHOLD=50` (line 685). c10's LED-reference page carries "24 V", "10 Hz", "(25 Hz)", "voltage" as native table content → NAMEPLATE 0.67 with ~170 OCR items; the dense override never ran. #2713's adversarial probes protect real dense nameplates via exactly this precedence, so the fix must be ratio-aware, not a blind reorder: at ≥50 items, holding NAMEPLATE should require plate vocabulary proper (`NAMEPLATE_KEYWORDS`) or a unit-field **ratio** (3 hits among 12 items = plate; 4 among 170 = table noise). **Fixture:** c10's capture from this run (real page, answer fragments included). **Effect:** with L1+L2, 12/12 cases enter the pipeline.

**L3 — Lift the Together vision timeout (stops losing computed answers; trivial knob).**
`mira-bots/shared/inference/router.py:209` — the together provider sets `timeout=30.0` hardcoded (the `_Provider` dataclass default is 60.0 at line 152; all three providers override to 30; there is no env knob). Vision/theory calls are Together-only — no cascade fallback exists for vision — and this run's successful theory calls measured 13.9–28.6s with 2 of 10 (c02/c05) crossing 30s → `together timeout after 30s` → "All providers exhausted" → honest fallback. #2804's 2000-token budget plus #2805's larger prompt consumed the old margin. **Fix:** `TOGETHERAI_TIMEOUT` env knob (default ~90s for the vision path) + compose mapping + `docs/env-vars.md` row (the enumerated compose env-block trap has bitten twice before). **Effect:** recovers the 2-answers-in-10 loss class; `SLOW_LLM_CALL` warnings become the new early indicator.
