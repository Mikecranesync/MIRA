## [Unreleased]

### Changed
- CI/CD required-checks accuracy (`.github/workflows/ci.yml`): the `CI` workflow no longer uses a `pull_request`-level `paths-ignore` (which left required checks stuck "Pending" and phantom-blocked docs-only PRs — the recurring `--admin` pain). It now always runs on PRs to main; a new `changes` filter job lets the heavy jobs (`test-unit`, `test-eval-offline`, `docker-build-check`, `ocr-recall-gate`, `simlab-gate`, `drive-pack-extract-tests`) self-skip on docs-only PRs (a skipped JOB reports "Success", so nothing blocks). Adds a single **`CI Gate`** aggregator job (`if: always()`) intended to replace the hand-picked required set so real correctness/security checks actually gate merges.
  - **Fail-closed evaluation** (no blanket "skipped == pass"): the `changes` job is in `CI Gate`'s `needs` and MUST equal `success` — a failed/omitted filter can never let the gate pass. Every always-run gated job (`actionlint`, `migration-order-check`, `lint-and-type-check`, `mira-hub-unit`, `mira-web-pack-tests`, `architecture-check`, `license-check`, `sast-semgrep`, `sast-bandit`) MUST equal `success`; `skipped`/`cancelled` are rejected. `test-unit` may be `skipped` ONLY when `changes` succeeded AND `changes.outputs.code == 'false'` (a genuine docs-only PR); when code changed it MUST equal `success`.
  - **Workflow correctness is gated:** `actionlint` moves in-tree as a `ci.yml` job included in `CI Gate` (and the standalone `actionlint.yml` is removed — it was never a required check), so a malformed workflow now fails the gate instead of merging unguarded. `actionlint` is NOT skipped on docs-only PRs.
  - **Fast correctness jobs still run on docs-only PRs** (never skipped): `lint-and-type-check`, `architecture-check`, `migration-order-check`, `license-check`, `sast-semgrep`, `sast-bandit`, `actionlint`. Only the six expensive jobs skip.
  - **Not in the gate yet** (stay visible; promote once de-flaked): `secrets-scan` (gitleaks flakes), `docker-build-check` (trivy CVE drive-by + ~6 min), and the heavy quality gates `simlab-gate` / `ocr-recall-gate` / `drive-pack-extract-tests` / `test-eval-offline`.
  - **Branch protection is UNCHANGED by this PR.** `CI Gate` ships as a non-required check first; requiring it (and dropping the standalone `mira-web pack tests`) is a separate deliberate admin step once it is proven green. (v3.183.1)
