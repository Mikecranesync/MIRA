| `PRINT_THEORY_VERIFY` | mira-bots ‚Äî when `1`/`true`/`on`, the print-theory cascade runs ONE self-verification pass after a non-empty draft (v3.178.0): the vision model re-reads the sheet against its own draft ‚Äî fixes misquotes to the printed text, deletes claims it cannot see printed, adds printed tiers/contacts/cross-refs it omitted on the asked device. | `PRINT_THEORY_SELF_CONSISTENCY` | mira-bots ‚Äî integer N (default `0`/off, or-form parse) for the FREE print-theory cascade (`engine.py::_grounded_print_reply`). When N>=2, the theory(+`PRINT_THEORY_VERIFY`) chain is sampled N times per turn and reconciled DETERMINISTICALLY to the medoid ‚Äî the reply that agrees most with the others by token-set Jaccard, no LLM judge ‚Äî which drops an idiosyncratic outlier (a one-sample fabrication, a minority coin-flip on a near-ambiguous reading) and keeps the majority reading (v3.179.0). Variance reduction toward a 9.0 single-round mean (ROUND5 Addendum 3: the residual to 9.0 is run variance on three cases; per-case best-of-three already measures 9.1). Any sample error falls through to the surviving samples; the turn is never eaten. Cost = N x the chain; the summed token usage across all samples is recorded into `printsense.interpret.pop_last_usage()`. N<2 = single pass, unchanged. Empty string (compose `${VAR:-}` shape) = off. Composes with `PRINT_THEORY_STYLE`/`PRINT_THEORY_FULL_RES`/`PRINT_THEORY_VERIFY`. |
| `LLM_REASONING_RETRY_MAX_TOKENS` | mira-bots ‚Äî token cap for the router's single reasoning-burn retry (v3.176.5). Reasoning models (Kimi-K2.6, LFM2.5) can consume the entire `max_tokens` budget on `<think>` chain-of-thought and return HTTP 200 with empty visible content (Tower OP round 4: 20/22 such calls); the router now strips `<think>` blocks from every reply and, when visible content is empty AND tokens were burned (cap hit / `reasoning_content` present / think markup), retries that provider ONCE with this cap before cascading on. Default **8192**; `0` disables the retry. Read as `int(os.getenv("LLM_REASONING_RETRY_MAX_TOKENS") or "8192")` ‚Äî `or`-form mandatory (compose `${VAR:-}` empty-string trap). |
