## Duplication / overlap flags

- **No duplicate implementations found** — the 07-09 discovery doc explicitly confirms this: reader, generator, live extractor, and store are each single, non-duplicated seams reused across PR-1→4 and the Print Translator. The two features (Print Translator vs wiring lane) are deliberately kept **architecturally separate** (print_translator.py docstring: "IS NOT the wiring DB lane") even though both consume the same `vision_worker`/`ELECTRICAL_PRINT` classification and the same `engine._extract_schematic` seam — this is intentional layering, not accidental duplication.
- **Docling/Tika ingestion migration** (unrelated PDF-text path, not wiring-specific) has ~7 open branches; newest is `fix/needs-ocr-tika-drain` (#2539) — flagged in the discovery doc as "coordinate, don't fork," not something to touch from the wiring/print-translator lane.
- **`feat/northwind-cv200-cloud-wiring`** branch name is a false positive for this domain — it's Ignition/Perspective cloud wiring, unrelated to electrical-print reading.
- Two doctrine-flagged (not accidental) tensions exist around mig-026's `ai_suggestions` requirement: PR-2 (`wiring_schematic_import.py`) and PR-4 (`wiring_intake.py`) both direct-INSERT LLM-derived rows as `proposed` instead of routing through `ai_suggestions`, each with an explicit docstring recording the decision — both point to the same unresolved bridge, issue **#2605**.
