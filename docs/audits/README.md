# Hub Page Audits

Append-only history of hub-page-audit runs. Generated by
`mira-hub/tests/e2e/hub-page-audit.spec.ts` (spec:
`docs/specs/enforcement-layer-spec.md` §4.1).

## How to read

Each file is `YYYY-MM-DD-audit.md`. The headline is `Score: N/M` — `N` routes
healthy out of `M` total. Below the score, a per-route table lists HTTP
status, paint time, body size, console errors, and exceptions.

## Regression detection

The audit script reads the most recent prior file and computes a delta. If
`N` drops, the report includes a "Regression detected" callout. Phase 1
(now → 2026-05-21) only logs the regression; Phase 2 will fail the deploy
job.

## Running locally

From the repo root:

```bash
cd mira-hub
HUB_URL=https://app.factorylm.com/hub npx playwright test tests/e2e/hub-page-audit.spec.ts
```

The report lands at `docs/audits/$(date +%Y-%m-%d)-audit.md`. Re-running the
same day overwrites the day's file (intentional — last run wins).

## Manually triggering on CI

```bash
gh workflow run enforcement-audit.yml --field run_audit=true
```

The job uploads `hub-audit-<run_id>.zip` as an artifact regardless of pass/fail.
