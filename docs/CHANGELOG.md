### v3.89.0 (2026-07-07) - fix(drive-commander): PowerFlex 525 P053 cross-ref gap (jitter-tolerant detector) + path-free grading reports (refs #2517)
- **Root cause (the real one, not the guessed one):** the scientific back-grade docked PowerFlex 525 to A/98.3 on **fault field accuracy** — only 4 of 6 gold fault→param links resolved. The prior finding blamed "P053 was never extracted"; **that was wrong.** P053 *is* in the pack (page-66 Basic Program grid). The bug was in `extractor._find_cross_refs`: `F100 [Parameter Chksum]` and `F109 [Mismatch C-P]` both say *"Set P053 [Reset To Defaults] to 2/3"* in their action text, but pdfplumber renders the bracket `[Reset` ~0.0003 pt *above* its own id `P053`, so a raw global `(top, x0)` sort placed the bracket *before* the id and the "id immediately followed by `[`" adjacency check silently dropped the reference (`F101` survived only because its tokens tied).
- **Fix:** `_find_cross_refs` now clusters the action band into visual LINES first (the same `_LINE_TOL` tolerance the rest of the extractor already uses) and scans each line in reading order — jitter-tolerant. With the fix `P053.related_faults = [F100, F101, F109]` and all 6 links resolve. Regression test added (`test_find_cross_refs_tolerates_subpixel_top_jitter`, crafted from the real token geometry — the old global-sort returned `[]`). The gold was completed (P053 added as a graded parameter with its fault links). The PF525 F081/F082/F083→C125 and PF40 F81→A105 headline links are unchanged.
- **Proof (grader measures, humans promote — no runtime pack change):** the **live** PF525 pack is UNTOUCHED and still carries the gap — it grades **A/96.9** against the now-complete gold (the gap was slightly undercounted before). The **regenerated staged candidate** (`tools/drive-pack-extract/candidates/powerflex_525/`, trust-status `beta`) grades **A/100**. The live pack reaches 100 only on a separate, human-gated **re-promotion**.
- **Also — path-free grading reports (repeatability):** `grade.py`/`report.py` embedded the manual's ABSOLUTE local path in the committed report (`manual.path` + `extraction_command`) — a machine-specific temp path is noise and a minor local-path leak in a "reproducible" artifact. `report.py` now stores the manual FILENAME only; `grade.py` reduces absolute-path argv tokens to basenames. Both existing committed candidate reports (PF525 new + PF40 regenerated) are now path-free; guard test added.
- **Scope:** grading tool + gold + candidate + docs only. **No runtime change** — the live `packs/powerflex_525/` and `packs/durapulse_gs10/` trees are untouched; no pack promoted. Real OEM manuals never committed. 93 tests green, ruff clean.

### v3.88.0 (2026-07-07) - feat(drive-commander): scientific drive-pack grading rubric (0-100 / A-F) + back-grade of all packs
- **What:** implements `Drive_Pack_Scientific_Grading_Spec` — a deterministic, weighted, numeric (0–100 + A–F) grader that replaces subjective review, on TOP of the existing five-layer harness (which stays the measurement engine). *No drive pack is promoted on subjective review alone. Grade before promote.* **Complements** the existing `beta`/`trusted` trust-status; does not replace it.
- **Rubric (`tools/drive-pack-extract/grading/scientific.py`):** 3 hard gates (schema validity, runtime compatibility — both proven by loading through the real `drive_packs/loader.py` — and provenance presence; any failure → grade **F**, not promotable) + 8 weighted scored categories (provenance 10, fault coverage/precision 20, fault field accuracy 20, param coverage/precision 20, param field accuracy 15, relationship accuracy 10, citation fidelity 15, safety/usability 10). The weights sum to 120, so the overall is a weighted **average** that **renormalizes over gradeable categories** — a pack with no gold set (categories 2–6 N/A) or no manual (citation fidelity N/A) is flagged **INCOMPLETE** and **not promotable** rather than silently scored zero. Grade bands A(92+)/B(82+)/C(70+)/D(50+)/F. Required outputs (spec): overall score+grade, category scores, critical failures, fault/param recall+precision, citation accuracy, missing evidence, promotion recommendation — as machine-readable JSON + human-readable Markdown. Reuses `gold_score`'s `_norm`/`_norm_set` so comparisons match the trust-status path exactly; never re-implements a measurement.
- **CLI (`grade_scientific.py`):** `--gold` is optional (a no-gold pack is graded on gold-independent categories only); exits non-zero when not promotable (a promotion gate). +15 tests (91 in the tool suite, all green; ruff clean) covering band boundaries, weighted-average-over-gradeable, N/A handling, hard gates, critical failures, the relationship-leak floor, and an integration grade of the committed PF40 candidate.
- **Back-grade published (`docs/drive-commander/scientific-grading/`, per the spec's Back-Grading Requirement):** **powerflex_40** (staged candidate) = **A / 100** (full gold + manual 22B-UM001J); **powerflex_525** (live) = **A / 98.3** — the grader precisely caught a real gap (only 4/6 fault→param links present; P053 never extracted); **durapulse_gs10** (live) = **D / 60 / INCOMPLETE, not promotable** — no gold set (coverage/accuracy N/A) AND its non-PowerFlex ID conventions (dotted `P09.03`, `CE10` faults) are flagged by the PowerFlex-oriented domain rules. GS10 not clearing the bar is the intended result: *a pack promoted before this bar existed does not automatically pass it.* Recommendations recorded: author a GS10 gold set + generalize the domain rules for GS10's conventions; extract P053 for PF525.
