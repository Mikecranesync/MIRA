## Part 2 тАФ FactoryLM Ignition Connector v0 (packaging plan)

**Principle:** v0 is a documented bundle of the existing path тАФ **no new connector system, no `.modl`,
no UDTs, read-only OT, fail-closed allowlist, garage untouched.**

### Package inventory (all files already on `main`)

**Edge (ships to the customer gateway):**
- `ignition/gateway-scripts/tag-stream.py` тАФ read-only Gateway Timer (browse + read + HMAC POST).
- `ignition/webdev/FactoryLM/api/connect/doPost.py` тАФ activation (`code тЖТ TENANT_ID + RELAY_URL`).
- `ignition/webdev/FactoryLM/api/tags/{collector.py,allowlist.py}` + `api/chat/signing.py` тАФ batch
  build + client-side allowlist + HMAC signing.
- `plc/ignition-project/NorthwindBottling/` тАФ reference Perspective project (rename per customer):
  views `ConvSimpleLive` / `MiraAsk` / `AnomalyCard` / `MaintenancePanel` / `Conveyor`; scripts
  `mira_chat`, `mira_diagnose` (+ `mira_diagnose_core`), `mira_setup` (tagтЖТrole mapping UI),
  `mira_tag_map`, `mira_signal_roles`, `mira_asset_config`.
- `docs/integrations/ignition-tag-collector.md` тАФ the collector contract.

**Cloud (already deployed тАФ customer does not install):**
- Claim: `mira-web /api/connect/{generate-code,activate,status}` + `plg_activation_codes`.
- Ingest: `mira-relay /api/v1/tags/ingest` + `auth.py` (HMAC) + `tag_ingest.py` (fail-closed allowlist).
- Hub: `/api/command-center/{commissioning,gateways,tree,display}`.

**Per-customer data (the only thing ops authors per site тАФ CV-200 is the template):**
- `tools/seeds/<customer>-hub.sql` (entities/equipment),
- `tools/seeds/approved_tags_<customer>.sql` (allowlist),
- `mira-hub/db/seeds/command_center_<customer>.sql` (display registration).

### Required config (`factorylm.properties` on the gateway тАФ placeholder names only)
`INGEST_URL` ┬╖ `TENANT_ID` (UUID) ┬╖ `MIRA_HMAC_KEY` (must equal the relay's `MIRA_IGNITION_HMAC_KEY`)
┬╖ `STREAM_TAG_FOLDER` (e.g. `[default]MIRA_IOCheck`) ┬╖ `STREAM_SOURCE_CONNECTION_ID` ┬╖ `STREAM_MAX_RETRIES`.
> The gateway property is `MIRA_HMAC_KEY`; the relay env is `MIRA_IGNITION_HMAC_KEY` тАФ **same value,
> different name**. Pull from `factorylm/stg` (or `prd`); never commit or print the value.

### Stranger-install guide (outline)
1. **Hub:** create customer/site/equipment тЖТ build the UNS node.
2. **Hub:** *Generate connect code* (`/api/connect/generate-code` тЖТ `MIRA-XXXX-XXXX-XXXX`).
3. **Gateway:** import the FactoryLM project folder; open ConnectSetup; paste the code тЖТ
   `/api/connect/activate` writes `TENANT_ID` + `RELAY_URL` to `factorylm.properties`.
4. **Gateway:** add `MIRA_HMAC_KEY`, set `STREAM_TAG_FOLDER` to the customer's tag root; map tagsтЖТroles
   with `mira_setup`.
5. **Ops:** author + apply `approved_tags_<customer>.sql` (staging тЖТ prod). Ingest is fail-closed тАФ
   nothing flows until approved.
6. **Gateway:** enable the tag-stream timer.
7. **Verify from anywhere:** Command Center тЖТ **Remote Commissioning** panel turns green
   (claimed тЖТ online тЖТ bound тЖТ approved тЖТ live тЖТ Ask-MIRA), or shows the exact next action.
8. **Register the live display** тАФ dev/staging: 8890 origin-root proxy; **production: dedicated
   FactoryLM origin (ADR-0024), e.g. `https://<customer-gateway>.factorylm-gateways.com`** тАФ never the
   raw gateway, never the 8890 proxy.

### Validation checklist (per install)
- [ ] Code activated (row in `plg_activation_codes`)
- [ ] Gateway online (Hub gateways probe)
- [ ] `approved_tags > 0` for the tenant
- [ ] `live_signal_cache` fresh under the asset UNS subtree
- [ ] Unknown tag returns `rejected:[{reason:not_allowlisted}]`
- [ ] Remote Commissioning panel green
- [ ] Ask-MIRA returns a grounded answer; `422 {"error":"uns_required"}` when no asset id

### Troubleshooting checklist
- Panel "not configured" тЖТ `TENANT_ID` / `MIRA_HMAC_KEY` missing in `factorylm.properties`.
- All tags rejected тЖТ allowlist not seeded / wrong `STREAM_TAG_FOLDER` (e.g. garage default) /
  normalizer mismatch.
- `401 auth_failed` тЖТ HMAC key mismatch or clock skew (┬▒300 s).
- Cache empty but timer running тЖТ folder has no leaf tags, or tags not allowlisted.
- Display blank тЖТ framed via raw gateway or 8890 in production (must be the dedicated origin).

### What still requires a human
Importing the project; setting `factorylm.properties` (incl. the HMAC value); tagтЖТrole mapping;
enabling the timer; authoring + approving the per-customer allowlist; provisioning the production
display origin.

### What later becomes a real `.modl`
`tag-stream.py` + the WebDev `connect`/`tags`/`chat` endpoints + a bundled settings/config UI тЖТ a
signed Ignition module with one-click install. **Explicitly out of v0** тАФ v0 is the project-import +
scripts bundle.
