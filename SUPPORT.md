# Getting support

Different problems go to different places. Pick the row that fits and you'll get help fastest.

| What you need | Where to go |
|---------------|-------------|
| **"How do I do X in the product?"** | In-app help: [app.factorylm.com/help](https://app.factorylm.com/help) |
| **A bug — something broken in the product** | [Open a GitHub issue](https://github.com/Mikecranesync/MIRA/issues/new?template=bug_report.md) |
| **A feature request** | [Open a GitHub issue](https://github.com/Mikecranesync/MIRA/issues/new?template=feature_request.md) |
| **A security vulnerability** | [SECURITY.md](SECURITY.md) — do **not** open a public issue |
| **A question about the code or architecture** | [GitHub Discussions](https://github.com/Mikecranesync/MIRA/discussions) |
| **Contributing to the project** | [CONTRIBUTING.md](CONTRIBUTING.md) |
| **Plant down, need help fast** | Email `mike@cranesync.com` with subject `URGENT — machine down` |
| **Sales, pricing, or commercial questions** | Email `mike@cranesync.com` |

## In-app help (fastest for product questions)

The fastest way to answer a "how do I…" question is the in-app help center at [`app.factorylm.com/help`](https://app.factorylm.com/help). It covers:

- A 5-minute getting-started walkthrough
- Feature guides for every screen (Assets, Work Orders, PM Schedule, Knowledge Base, etc.)
- The 10 questions we hear most often
- Keyboard shortcuts and tips
- Contact details with response-time expectations

If the answer isn't there, that's a bug in the help — please email so we can fix it.

---

## GitHub Issues

GitHub Issues are for **bugs and feature requests** in the open-source code, not for product support. Before opening one:

1. Search existing issues — yours may already be filed.
2. Check [`docs/known-issues.md`](docs/known-issues.md) — known limitations live there.
3. Reproduce on the latest `main` if you can.
4. Use the issue template — it asks for the right context.

Issues that don't follow these rules may be closed with a pointer to the right place. Not personal — just keeping signal high.

---

## GitHub Discussions

For everything that isn't a bug, feature request, or security report — design discussions, "is this the right way to do X," "I'd like to understand why Y was chosen" — use [GitHub Discussions](https://github.com/Mikecranesync/MIRA/discussions). Lower bar to entry than a PR or an issue.

## Direct email

`mike@cranesync.com` reaches the lead maintainer (Mike Harper) directly. Use this for:

- Anything sensitive (security, customer data, conduct concerns)
- Sales and pricing
- "Plant is down, need help fast" — put `URGENT` in the subject line

Response time:
- **Urgent:** same-day during US business hours (Monday–Friday, 8 AM – 6 PM ET).
- **Normal:** within 1 business day.

## What we **don't** do

- We don't do live chat support.
- We don't have a status page yet (`status.factorylm.com` is reserved but unbuilt).
- We don't accept feature requests via Twitter or LinkedIn DMs — please use a GitHub issue or email so we don't lose them.
